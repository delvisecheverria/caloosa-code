QA ENGINEER PROGRAM

Weeks 1–4: QA Fundamentals & Manual Testing

Content:

  • Introduction to the Software Development Life Cycle (SDLC)
  • Types and levels of testing, QA roles
  • Test case design and execution
  • Test case management using AIO Test / TestRail
  • Bug reporting and management in Jira
  • Hands-on practice with a sample web application and full traceability

Objectives:

  • Understand the software development lifecycle and QA’s role in it
  • Differentiate between testing types and levels
  • Design, execute, and manage manual test cases
  • Report bugs with clarity and reproducibility
  • Demonstrate traceability from requirements to bugs

Weeks 5–10: Web & Mobile Test Automation

Content:

  • Web Automation (Weeks 5–7):
  • Selenium + C# fundamentals
  • Test framework structure using NUnit or MSTest
  • DOM interaction, locators, and Page Object Model
  • BrowserStack, GitHub integration
  • Mobile Automation (Weeks 8–10):
  • Mobile testing fundamentals (iOS / Android)
  • Appium setup for local and emulator testing
  • Framework structure for mobile automation
  • Locator strategies for mobile apps

Objectives:

  • Build a basic test automation framework with Selenium and C#
  • Automate functional test cases using DOM locators
  • Apply Page Object Model and maintain scripts efficiently
  • Configure Appium and run tests on real devices or emulators
  • Extend automation skills to mobile testing scenarios

Weeks 11–14: API Testing + Databases

Content:

  • SQL fundamentals: SELECT, INSERT, UPDATE, DELETE
  • Queries for test data validation using DBeaver
  • REST API fundamentals: JSON, HTTP methods, Body
  • Endpoint functional testing with Postman

Objectives:

  • Write and run SQL queries to validate data
  • Test REST API endpoints for functionality and accuracy
  • Create Postman collections, use environments, and automate assertions

Weeks 15–17: Performance Testing

Content:

  • Types of load, stress, and endurance tests
  • Scripting with JMeter
  • Load generation with OctoPerf
  • Result analysis and performance monitoring

Objectives:

  • Understand performance testing methodologies
  • Create JMeter scripts for load and stress testing
  • Generate realistic load scenarios using OctoPerf
  • Analyze and interpret performance metrics
  • Provide recommendations based on results

Weeks 18–20: Capstone Project & Career Prep

Content:

  • End-to-end QA project using real-world scenarios
  • Application of manual, API, and automation testing
  • Building final QA portfolio and presentation
  • Resume and interview preparation

Objectives:

  • Apply all QA concepts in a complete project
  • Build a professional QA portfolio
  • Prepare for job interviews with confidence

Months 6–8: Internship / First QA Job

Content:

  • Hands-on QA experience with live projects
  • Work with teams and industry tools
  • Mentorship and feedback sessions

Objectives:

  • Gain real industry experience
  • Build confidence in professional environments
  • Secure your first QA position
